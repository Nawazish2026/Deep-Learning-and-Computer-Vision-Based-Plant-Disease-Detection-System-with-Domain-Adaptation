SLIDE 2: INTRODUCTION
===============================================================================
- AI Powered Plant Disease Detection System  is a full-stack
  ML-powered web application for real-time plant disease diagnosis using leaf images.
- Domain: AI / Machine Learning / Deep Learning / Computer Vision / Full-Stack Web.
- Real-world use case: A farmer in the field photographs a diseased plant leaf
  on their phone and receives an instant, accurate diagnosis in seconds.
- Importance: India loses ~15–20% of crop yield annually due to undetected plant
  diseases. Early detection is critical for food security.
- The system is designed to bridge the gap between controlled lab AI models
  and messy real-world field conditions ("Domain Gap" problem).
- Goes beyond classification — includes a Knowledge-Grounded RAG chat system
  to provide verified treatment recommendations from agricultural manuals.

SLIDE 3: PROBLEM STATEMENT
===============================================================================
- Existing ML plant disease models are trained exclusively on clean, studio-lit
  laboratory images (PlantVillage dataset) and fail drastically in real field use.
- Farmers and agronomists cannot access expert consultation in rural areas quickly.

- Target users affected: Small and marginal farmers (70%+ of India's farming

  (fine-tuned), which doesn't qualify as "custom" for academic Major Project standards.

SLIDE 6: DATASET PREPARATION
===============================================================================
- PRIMARY DATASET: PlantVillage (Color) — Hughes & Salathé (2015).
  URL: https://www.kaggle.com/datasets/emmarex/plantdisease
  - 54,305 color images, 38 classes across 14 plant species.
  - Covers: Apple, Tomato, Corn, Potato, Grape, Strawberry, etc.

- SECONDARY DATASET: PlantDoc — Singh et al. (2020).
  URL: https://github.com/pratikkayal/PlantDoc-Dataset
  - 2,231 images mapped / 27 classes with complex real-world backgrounds.
  - Images include dirt, shadows, multiple leaves, and outdoor lighting.

TRAIN/TEST SPLIT:
  - Phase 1 (Base Training): 80% Train (43,444) / 20% Val (10,861) from PlantVillage.
  - Phase 2 (Mixed Fine-Tuning): Combined 56,536 images → 80% Train / 20% Val.

DATA COLLECTION METHOD:
  - torchvision.datasets.ImageFolder for automatic folder-based class loading.
  - Custom MappedPlantDocDataset class written to remap PlantDoc folder names
    to exact PlantVillage class indices (26 out of 27 mapped; 1 class skipped).

PREPROCESSING:
  - Image resize: 224x224 pixels (standard for CNN input).
  - Normalization: Mean=[0.485, 0.456, 0.406], Std=[0.229, 0.224, 0.225] (ImageNet standards).

DATA AUGMENTATION (Applied during training only):
  - RandomHorizontalFlip — simulates left/right leaf views.
  - RandomRotation(±20°) — simulates angled photography in the field.
  - ColorJitter (brightness=0.2, contrast=0.2, saturation=0.2, hue=0.1) — simulates
    different lighting conditions (morning sun, overcast, shade).

SLIDE 7: METHODOLOGY
===============================================================================
STEP-BY-STEP SYSTEM WORKING:

PHASE 1 — Base Model Training (train_agrinet.py):
  1. Load PlantVillage (38-class) dataset via ImageFolder.
  2. Apply augmentations to training split; standard normalize to validation split.
  3. Initialize AgriNet (custom 4-block CNN with Inverted Residual Blocks).
  4. Train for 10 epochs with Adam optimizer (LR=0.001).
  5. ReduceLROnPlateau scheduler automatically adjusts LR when val_loss stagnates.
  6. Save best model weights to agrinet_v1.pth (based on minimum val_loss).

PHASE 2 — Mixed Domain Fine-Tuning (mixed_finetune_agrinet.py):
  1. Load agrinet_v1.pth (98.75% accuracy on PlantVillage).
  2. Load PlantDoc dataset; apply custom mapping to match 38-class label structure.
  3. Concatenate PlantVillage + PlantDoc using ConcatDataset.
  4. Fine-tune entire model (all layers) with tiny LR=0.0001 (10x smaller).
  5. Train for 10 more epochs; save best mixed-domain model → agrinet_mixed_v1.pth.

ML PIPELINE FLOW:
  Image Input → Resize(224x224) → Normalize → AgriNet Forward Pass →
  Softmax → Top-1 Predicted Class + Confidence Score → Return Result

SLIDE 13: IMPLEMENTATION DETAILS
===============================================================================
MODULE 1: AgriNet CNN Architecture (train_agrinet.py)
  - AgriBlock class: Implements the 3-phase Inverted Residual block.
    Phase 1 (Expand): 1x1 pointwise conv to expand channels by expansion factor.
    Phase 2 (Filter): 3x3 depthwise conv operating on each channel independently.
    Phase 3 (Project): 1x1 pointwise conv to compress to output channels.
  - Residual connection applied only when stride=1 AND in_channels == out_channels.
  - AgriNet backbone: Stem (3→32) → 4 AgriBlocks (32→64→128→256→512) →
    Global Average Pooling → Dropout(0.3) → Linear(512, 38).

MODULE 2: Mixed-Domain Fine-Tuning (mixed_finetune_agrinet.py)
  - MappedPlantDocDataset: Custom Dataset class iterating PlantDoc folders and
    remapping 26 folder names to their exact PlantVillage class indices via
    PLANTDOC_TO_PV_MAP dictionary.
  - ConcatDataset joins 54,305 PlantVillage images + 2,231 PlantDoc images = 56,536.
  - Fine-tuning uses LR=0.0001 (10x smaller than base training) to avoid
    catastrophic forgetting of the 98.75% accuracy already achieved.

MODULE 3: Evaluation (evaluate_agrinet.py)
  - Runs the mixed model over the full validation set.
  - Outputs sklearn classification_report: per-class precision, recall, F1.
  - Generates a 38x38 seaborn heatmap (Confusion Matrix) saved as PNG.

MODULE 4: Field Test (test_field_data.py)
  - Loads 6 random PlantDoc test images and runs the 38-class model on them.
  - Displays raw field image + Top-3 predicted classes side by side for visual proof.

MODULE 5: Backend API (FastAPI) [Planned — Ankit's responsibility]
  - POST /auth/register, POST /auth/login → JWT token authentication
  - POST /predict → Image upload, model inference, return disease + confidence
  - GET /history → Returns user's past predictions from PostgreSQL

MODULE 6: RAG Engine [Completed — Sahil's responsibility]
  - ingest.py: Loads PDF manuals, splits text into chunks, generates local sentence-transformer embeddings (all-MiniLM-L6-v2), and saves to local FAISS index.
  - chat.py: Loads FAISS index, retrieves top-3 matching context chunks, and queries Groq API (llama-3.1-8b-instant) to generate grounded answers.

MODULE 7: Frontend [Planned — Nawazish's responsibility]
  - Next.js pages: Landing, Login, Dashboard, Chat.
  - Glassmorphism design system with Framer Motion transitions.
  - PDF Report generator showing prediction + heatmap.
